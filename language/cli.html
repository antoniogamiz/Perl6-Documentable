Perl6::Documentable provides you with a easy-to-use CLI application. This is useful if you only want to generate the static files as serve them with nginx or similars.

It has a lot of options to let you customize the generation process as much as you want. Let's see what's available.

Options

documentable setup

This will download a bunch of files in your current working directory. You can check those files here.

You do not need to use these exact files. You can modify them as you want, except for the files under template dir.

documentable start

This will start the HTML generation process. See the parameters below to customize it.

documentable update

This one is very importante. It will update your HTML collection without regenereting each and every file.

documentable clean

This will delete the directories and files generated by documentable setup.

Parameters

--topdir=<Str>

Root directory where all your documentation is stored.

Default: "doc".

--conf=<Str>

Path to the configuration file (see configuration-file).

If a file called "config.json" is not found at the same directory where Perl6::Documentable was called, the default config file will be used instead.

Default: "config.json".

-v, --verbose

It True, progress information will be displayed.

Default: False.

-c, --cache

If True, Pod::To::Cached will be used to load the pods and create a cache. Recommended.

Default: True.

-p, --primary

If True, HTML files associated with Perl6::Documentable::Primary objects will be generated. That means, all your collection will be mapped to HTML, generating one HTML file by each pod block found.

Default: False.

-s, --secondary

If True, HTML files associated with Perl6::Documentable::Secondary objects will be generated. Those files comes from grouping those objects by name.

Default: False.

--search-index

If True, a search index will be generated to enable search capabilities in the site.

Default: False.

-i, --indexes

If True, index pages will be generated to group all HTML pages by Kinds and categories. See configuration-file.

Default: False.

-t, --typegraph

If True, TypeGraph representations will be generated. See typegraph.

The representations will be cached when possible.

Default: False.

-f, --force

If True, TypeGraph representations will be always generated. See typegraph.

Default: False.

--highlight

If True, block codes will be highlighted. See highlighting.

Default: False.

--typegraph-file=<Str>

Path to the TypeGraph file. See typegraph.

Default: "type-graph.txt".

--highlight-path=<Str>

Path to the directory containing the highlighter files. Internal use.

Default: "./highlights".

--dirs=<Str>

Comma sepparated values to specify the directories where Perl6::Documentable will look for documentation. Example: "--dirs=Language,Type".

Default: DOCUMENTABLE-DIRS.

-a, --all

Equivalent to specify -t -p -s -i --search-index

Default: DOCUMENTABLE-DIRS.

Examples

Suppose you only want to process your pod collection to check the syntax in you pod files. You only to execute:

documentable start

Or maybe you only want to generate the primary files:

documentable start -p

Or the search index:

documentable start --search-index

And what if I have a different doc structure? For instance:

documentation
└── Potatoes
    ├── Potato1.pm6
└── Carrots
    ├── Carrot1.pm6

You only need to specify the correct dirs to Perl6::Documentable:

documentable start --topdir=documentation --dirs=Potatoes,Carrots

The command used to generated the documentation for Perl6::Documentable is:

documentable start --topdir=docs -v -a --highlight --highlight-path="/highlights"
